Design: Dual API Mode, Security & Bonus Features

The final pieces — how both auth modes work and what ships in v1

Dual API Mode

Provider routing in background.js
Mode A: User's own key
1. User enters API key in settings
2. Key stored in chrome.storage.local
3. background.js reads key, calls LLM directly
4. Request: fetch('https://api.openai.com/v1/...')
Works for: OpenAI, GLM (any provider with a key)
Mode B: Backend proxy
1. User signs up / logs in via popup
2. Auth token stored in chrome.storage.local
3. background.js calls your API with token
4. Your server proxies to LLM, returns result
Enables: usage tracking, rate limits, billing, SaaS
Routing logic: Settings include an authMode field: 'user_key' or 'backend'. background.js checks this on every request and dispatches to the appropriate provider adapter. If 'user_key' but no key is saved, shows a notification to open settings.

Security Model

How secrets and data are handled
✓
API keys never touch the DOM
Stored in chrome.storage.local, read only by background.js. Content scripts never see them. Popup masks the key field.
✓
All API calls from background service worker only
Content scripts send messages via chrome.runtime.sendMessage. Background worker makes the actual fetch. Keys never exist in page context.
✓
XSS prevention
All injected UI uses document.createElement() — no innerHTML with user data. Generated text inserted via DOM text nodes.
✓
Minimal permissions
Manifest requests only: storage, activeTab, host permissions for the 4 platforms. No broad <all_urls> access.
✓
No auto-submit
Extension only inserts text into fields. Never triggers form submission, clicks buttons, or takes any action on behalf of the user.

Bonus Features (v1)

Ships alongside core features
⌨ Keyboard shortcut
Ctrl/Cmd + Shift + A opens the popover on the nearest editable field. Registered via chrome.commands in manifest. Works without clicking the AI button.
📋 Generation history
Each generated output saved to chrome.storage.local with timestamp, platform, and action type. Accessible from popup. "Reuse" inserts a previous output. Capped at 100 entries, FIFO.
🎨 Tone switch in popover
Tone chips are already in the popover design. Clicking a different tone re-runs the generation with the new tone applied. No page reload needed.
