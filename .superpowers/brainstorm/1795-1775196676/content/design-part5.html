Design: Platform Detection & Prompt Engine

Robust detection across 4 platforms, and how prompts get built

Platform Detection

Two-layer detection: URL first, DOM confirmation second
Layer 1: URL matching
linkedin.com → 'linkedin'
facebook.com → 'facebook'
twitter.com → 'x'
x.com → 'x'
reddit.com → 'reddit'
Runs once on page load. Stored for session.
Layer 2: DOM confirmation
data-testid, role, aria-label
contenteditable="true"
Known structural patterns
Placeholder text heuristics
Runs via MutationObserver. Confirms fields exist.
Why two layers: URL tells us which platform config to load. DOM confirmation tells us where to inject. This avoids running LinkedIn selectors on Reddit or vice versa. The MutationObserver only watches for editable fields using platform-specific selector maps.

Prompt Engine

How buildPrompt() assembles the final message
System prompt (always included)
"You are a human-like social media writer. Write naturally. Avoid robotic phrasing like 'Great post!' or 'I completely agree.' Add perspective, not summary."
Dynamic layers merged on top:
+ Platform personality
"You're writing for LinkedIn. Be insightful, professional, not corporate boilerplate."
+ Task instruction
"Generate a comment that adds a new perspective or real-world example."
+ Tone modifier
"Tone: witty. Use light humor if it fits naturally."
+ Context payload
"Post by Sarah Chen: 'Every feature request looks essential...'"
Output rules (appended to every prompt): No hashtags unless requested. No emojis unless casual tone. Slightly imperfect human tone. Never use filler phrases like "As a..." or "In my opinion...".
